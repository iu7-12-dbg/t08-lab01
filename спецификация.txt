Реализация алгоритма поиска пути А*.
В качестве навигационной сетки используется поле из MxN клеток. Перемещаться по полю можно по вертикали и по горизонтали. Для каждой клетки определен вес - стоимость перемещения в эту клетку, заданная вещественным числом. Если вес - отрицательное число, клетка считается непроходимой и не учитывается при построении пути.
Путь представляется в виде последовательности клеток, по которым необходимо перемещаться, чтобы попасть из клетки стартовой клетки в конечную. Путь включает в себя стартовую и конечную клетки. Если стартовая и конечная клетки совпадают, путь состоит только из одного элемента. Если построить путь между клетками невозможно, путь не содержит элементов.

Клетки поля реализуют интерфейс ICell, который определяет свойства Weight (возвращающее вес клетки в виде вещественного числа) и Coordinates (возвращающее структуру, содержащие координаты X,Y клетки). Алгоритм поиска пути реализует интерфейс IPathFindingAlgorithm, который определяет метод Process(ICell[,] CellMap, Coordinates start, Coordinates end) (возвращающий путь в виде последовательности клеток поля).

Тесты покрывают следующие сценарии:
	сравнение координат: (5,5) == (5,5), (0,0) < (5,5) и т.п.
	ожидаемые исключения при выполнении поиска: NullReferenceException при передаче null в качестве поля; IndexOutOfRangeException при передаче координат, находящихся за пределами поля
	проверка на содержание в пути начальной и конечной вершин
	проверка на равенство координат начальной и конечной вершин при построении пути из начальной клетки в неё же
	проверка на отсутствие элементов при построении пути в недостижимую клетку
